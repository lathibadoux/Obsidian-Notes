*  **Struct**
	* Keys are rigidly defined
	* Compression is good!
	* Values can be any type
* **Map**
	* Keys are loosely defined
	* Compression is okay!
	* Values all have to be the same type
* **Array**
	* Ordinal
	* List of values that all have to be the same type
	* Can actually nest Struct and Map in Array
