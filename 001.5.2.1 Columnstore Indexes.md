* The most powerful step is the **compression**.  Compression will create the dictionary to associate a string with an integer.  Reduces the size of the data and increases the performance.

## LoB Page
* Different from Rowstore data pages.
* Segment Header - metadata information about the column stored in this page.
	* Maps to a **Dictionary Page**
* Data Stream - Dictionary IDs
![[LOBpagevsdatapage.jpg]]
* Must define if the Columnstore index is Clustered or Non-clustered before the LoB page can be created.
* Clustered columnstore index will replace the old clustered rowstore index.  
* Non-clustered columnstore index just creates an additional structure.
